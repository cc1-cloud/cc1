Virtual machine creation form.

VM consists of base components:

  • image - image file containing VM's operating system, installed software and all configuration. There are default public images available as well as ones created by users (private or group). Images can be managed via Images section.
  • hardware template - VM's hardware parameters like CPU and RAM,

You can improve usability of the created VM, attaching some of the available resources to it:

  • IP address,
  • disk volume.

These resources are managed via VM Resources section.

Machines created using this form are accessible via Machines->Show virtual machines.

If VM isn't started from image configured so that access to it is enabled, you need either to enable VNC on that machine (only when starting), or attach public IP address to it (whenever).

Machine started from images with contextualisation can have its system user and password set via web interface. Its owner can also inject public key into it to enable login via SSH.
