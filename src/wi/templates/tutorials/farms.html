Farms are automaticaly created clusters of virtual machines with preconfigured batch system.

The functionality to create the preconfigured farm of VMs is provided to perform large scale processing. The farm can be created with any VM image with contextualization installed.

Every farm consists of:

  • a single head node (HN)
  • a specified number of worker nodes (WNs)

All are created with the same VM image.

The HN plays the role of NFS server and the batch system controller (if the batch system is installed). No central authentication is required. All users present on a base VM image appears as local users on HN and every WNs since they are created from the same VM image. They share the same /home directory exported from HN to all WNs.

The farm is created in two main steps:

  • In the first automatic step paswordless access from the HN to WNs is enabled.
  • In the second step configuration script is executed on HN to mount /home directory on all WNs.

More about farms you can read in the CC1 User Guide.
